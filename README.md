# ACTIVATION BOT v1

Первая рабочая версия игрового бота для реалити ACTIVATION.

## В первой версии

### Игрок
- регистрация через /start;
- автоматическое создание личной ветки игрока в закрытом админ-чате;
- отправка сообщений игрока в его ветку;
- ответы ведущей из ветки обратно игроку;
- меню: актуальное задание, прогресс, рейтинг, достижения;
- получение новых заданий;
- получение красивой карточки при начислении баллов;
- получение секретных миссий.

### Ведущая
Основной аккаунт ведущей не регистрируется как игрок. По `/start` и `/admin` открывается только панель ведущей.

В панели:
- создать новое задание;
- ввести название;
- ввести текст;
- назначить количество баллов;
- подтвердить и отправить всем зарегистрированным игрокам;
- посмотреть игроков;
- посмотреть рейтинг.

Дополнительно:
`/points USER_ID 100 причина` — начислить баллы.
`/secret USER_ID 200 Название | Описание` — выдать секретную миссию.

## Railway

ADMIN_CHAT_ID=-1004379372751
ADMIN_IDS=372122727
DB_PATH=/data/activation_bot.db

Создайте Railway Volume с mount path:

/data

База будет храниться в:

/data/activation_bot.db

Это позволяет сохранять игроков и баллы между redeploy.

## Важно

В закрытой админ-группе должны быть включены Topics.
ACTIVATION Bot должен быть администратором и иметь право управлять темами.

## Изменение v1.1
- аккаунт из `ADMIN_IDS` больше не создаётся как игрок;
- ведущая не попадает в рейтинг;
- ведущая не получает игровое меню;
- личка бота у ведущей используется только как админ-панель;
- путь игрока удобно тестировать с отдельного Telegram-аккаунта.


## Релиз v1.2 — вход в игру
- новый игрок по `/start` сначала видит атмосферное вступление;
- показываются 4 уровня: ЛИЧНОСТЬ → ВИДИМОСТЬ → ВЛИЯНИЕ → МАСШТАБ;
- регистрация происходит только после нажатия кнопки `🔥 ВОЙТИ В ИГРУ`;
- после входа создаётся персональная ветка игрока в админ-чате;
- уже зарегистрированный игрок по `/start` сразу получает игровое меню;
- ведущая по `/start` видит только панель ведущей.


## Релиз v1.3 — карта игрока
После кнопки `🔥 ВОЙТИ В ИГРУ` бот проводит игрока через онбординг:

1. Имя.
2. Чем занимается.
3. Точка А.
4. Цель на 21 день.
5. Фотография.

После этого:
- профиль сохраняется в постоянной базе;
- автоматически создаётся персональная карта игрока с его фото;
- карта отправляется игроку;
- карта дублируется в личную ветку игрока для ведущей;
- только после завершения онбординга создаётся личная ветка;
- открывается игровое меню.

Для уже существующей базы добавлена безопасная миграция новых полей профиля.


## Релиз v1.4 — утверждённая визуальная система ACTIVATION
- стартовая карта игрока переведена в светлый premium luxury-tech стиль ACTIVATION;
- холодный серебристо-белый фон;
- стеклянные панели и cyan/turquoise акценты;
- обязательное квадратное окно с автоматической вставкой фото;
- стартовая карта содержит: имя, направление, уровень 01, 0 баллов, точку А и главную цель;
- эта визуальная система зафиксирована как база для следующих карточек: баллы, задания, достижения, секретные миссии, уровни и прогресс.


## Релиз v1.5 — master-template карты игрока
- согласованный макет ACTIVATION добавлен в проект как `assets/player_card_template.png`;
- бот больше не рисует дизайн карты с нуля;
- карта генерируется поверх утвержденного шаблона;
- автоматически вставляются: фото, имя, Telegram username, направление, точка А, цель, дата старта;
- уровень на старте: 1 / ЛИЧНОСТЬ;
- баллы и дни в игре на старте: 0.


## Релиз v1.6 — постоянный игрок + кириллица

### Постоянное состояние игрока
- `tg_user_id` используется как постоянный уникальный ID;
- повторный `/start` не создаёт нового игрока;
- удаление чата с ботом и повторный вход не сбрасывают данные;
- существующая личная ветка не создаётся заново;
- баллы, уровень, профиль и история остаются в базе;
- незавершённая регистрация продолжается, а не создаётся заново.

### Кириллица на карточках
- в `assets/` добавлены `DejaVuSans.ttf` и `DejaVuSans-Bold.ttf`;
- `cards.py` использует только эти встроенные шрифты;
- русский текст больше не зависит от системных шрифтов Railway.


## Релиз v1.7 — адаптивный текст + удаление игрока

### Карточка
- длинный текст автоматически переносится;
- слишком длинный текст автоматически уменьшается;
- длинные слова без пробелов тоже безопасно разбиваются;
- `Точка А` и `Главная цель` больше не выходят за границы блоков.

### Панель ведущей
- в разделе `Игроки` появилась кнопка `🗑 Удалить игрока`;
- перед удалением есть подтверждение;
- удаляются профиль и связанные игровые данные;
- личная Telegram-ветка игрока сохраняется, чтобы не терять историю общения.


## Релиз v1.8 — карточка прогресса
- добавлен master-template `assets/progress_card_template.png`;
- кнопка `🎮 Мой прогресс` теперь отправляет красивую карточку в стиле ACTIVATION;
- динамически подставляются уровень, баллы, цель по баллам, день и серия;
- визуальная система едина со стартовой картой игрока.
